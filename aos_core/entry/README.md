# Entry / Bootstrap Runtime

This core gate controls the first touch of AOS.

It is not a file scanner. It does not understand project content. Its job is to
force the correct sequence before any normal answer or execution:

1. Operating Identity
2. Intent Frame
3. Surface Resolution
4. Truth Requirement
5. Artifact Cockpit
6. Truth Sufficiency
7. Response Delivery

File Intelligence is allowed to be the first operation only when the user
explicitly asks for `inspect`, `doctor`, or file/artifact diagnostics. In normal
conversation, File Intelligence is a cockpit after identity/intent/surface/truth,
not the entry gate.
