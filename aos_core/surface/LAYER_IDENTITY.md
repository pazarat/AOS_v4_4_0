# Surface Layer Identity

Surface is the visible-answer boundary and cockpit visibility controller.

## Purpose

Bind each request to what the answer may visibly discuss while preserving governed internal awareness.

## Not This

- Not a narrow route tunnel.
- Not a replacement for File Intelligence.
- Not a replacement for Truth.
- Not a reason to ignore Workshop or AOS lenses when they are silently relevant.

## Public Interface

- `SurfaceResolver.resolve(intent_frame)`
- `SurfaceResolver.bind(intent_frame)`

## Internal Responsibilities

- Decide visible surface: AOS, Workshop, or active project payload.
- Define allowed silent scopes.
- Define hidden scopes.
- Provide response rules and forbidden exposure.
- Protect project-facing answers from AOS/Workshop leakage.

## Governance Rules

```text
Intent selects visible surface, not a blind evidence tunnel.
Artifact Cockpit performs governed 360-degree inspection inside allowed scopes.
Final answer stays focused on the user intent and visible surface.
```

## Failure Signals

- Project answer exposes AOS internals without user request.
- Workshop standards are treated as project payload evidence.
- Intent sends the model into a single blind path.
- Response is based on partial truth when cockpit evidence was available.
