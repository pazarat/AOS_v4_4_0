# Kernel Layer Identity

The Kernel is the AOS control plane. It orchestrates layers; it does not replace them.

## Purpose

Coordinate identity, intent, surface, workspace, file intelligence, strategy, planning, policy, delivery, evaluation, maintenance, and audit.

## Operating law

The kernel must build operation packets from layered evidence, not from direct model intuition.

## Required flow

```text
boot identity
→ resolve intent
→ bind visible surface
→ inspect allowed scopes through File Intelligence
→ build operation packet
→ enforce runtime and response contracts
→ synthesize focused answer
→ audit receipt
```

## Failure signal

If the kernel routes from intent directly to a narrow path without allowed-scope cockpit inspection, AOS becomes blind and must be repaired.
