# فتحة المشروع النشط

`workshop/active_project/` فتحة تشغيل ثابتة داخل بيئة المهندس، وليست مشروع المستخدم.

المكان الافتراضي الوحيد لتحميل مشروع المستخدم هو:

```text
workshop/active_project/PROJECT/
```

القواعد:
- إذا كان `PROJECT/` فارغًا، فلا يوجد مشروع حالي محمّل.
- في حالة الفراغ، يجيب المهندس بوضع تكوين مشروع أو عرض مهاراته في بناء/تنظيم المشروع، ولا يقيّم AOS أو ورشة العمل كمشروع.
- `00_PROJECT_LOCAL_TRUTH/` حقيقة محلية داعمة للمشروع، وليست payload مشروع.
- `_aos_runtime_project_state/` حالة تشغيل، وليست payload مشروع.
- أي ملفات مشروع، مستودع قديم، وثائق، أو baseline جديد يجب أن توضع داخل `PROJECT/`.
- AOS وورشة العمل وهذه الفتحة ليست مشروع المستخدم.


## قاعدة عدم تثبيت اسم المشروع

لا يسجل AOS اسم أي مشروع داخل `PROJECT/` كحقيقة ثابتة. الاسم والبنية والحقيقة الخاصة تُكتشف من الحمولة المحملة داخل البوابة.

المسار الثابت هو البوابة فقط:

```text
workshop/active_project/PROJECT/
```

كل ما بداخلها قابل للتغيير حسب المشروع.
