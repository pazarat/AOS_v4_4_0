# Project Local Standards Generation Canon

The workshop prevents every project from rebuilding general standards from zero. The active project specializes only what applies.

## Flow

1. Detect project type, maturity, constraints, and delivery mode.
2. Select applicable workshop standards.
3. Compare project reality against the standards.
4. Produce local standard candidates.
5. Ask for or record acceptance where needed.
6. Treat accepted local standards as project truth for execution.

## Examples

A software project may specialize stack, architecture, DDD, database, migration, API, realtime, frontend, testing, deployment, and security rules. A research project may specialize source quality, citation, methodology, evidence grading, and review rules. A writing project may specialize audience, voice, structure, plot, doctrine, or editorial constraints.
