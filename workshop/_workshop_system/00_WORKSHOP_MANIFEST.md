# Workshop Manifest — General Truth Domain

The Workshop is the general truth and standards domain of the engineer environment. It is not the active project, not the AOS runtime, and not a project-specific memory.

Its role is to provide the universal method lens for building, evaluating, repairing, and maturing any project: software, research, writing, education, business operations, religious content, mixed systems, or scattered legacy work.

## Laws

- Workshop standards are general truth, not automatic local project truth.
- A standard applies only after intent, project type, project maturity, and local project truth are considered.
- Workshop can expose gaps, contradictions, missing dimensions, and candidate local standards.
- Execution must obey accepted local project truth, not generic best practice alone.
- No single documentation structure is mandatory. Parent/child is one relationship pattern, not a universal requirement.

## General-to-local flow

Universal standard → project reality comparison → missing dimensions → local standard candidate → accepted local project truth → execution.
