# Project Structure Discovery Canon

The engineer must not assume that every project uses parent/child documentation, PRD trees, modules, or software folders.

## Relationship patterns

The project may contain any mixture of:

- parent/child,
- owner/dependent,
- index/detail,
- chapter/section,
- source/analysis/report,
- workflow/use-case,
- code/module/test,
- standard/local-standard,
- scattered legacy material with no clear owner.

## Rule

Discover the actual structure from files and user intent. If no structure exists, enter formation or recovery mode and create a baseline proposal. Parent/child routing is useful when present, but never a requirement for understanding a project.
