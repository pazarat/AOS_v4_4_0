# Goal Runtime Layer Identity

Goal Runtime is the work-control capability for AOS. It is not the reasoning identity, not file intelligence, and not the truth runtime. Its purpose is to prevent repeated patch loops by turning high-impact work into a tracked goal with evidence, attempts, tests, and closure criteria.

## Philosophy

AOS must not keep repairing the same symptom through version after version without proving that the goal was actually reached. When a problem survives several fixes, Goal Runtime forces a pause: record the target, compare attempts against the target, identify the root cause, and close only when evidence proves the condition.

## Role with other layers

- Entry Runtime ensures the correct sequence.
- Truth Runtime defines what must be true.
- File Intelligence discovers evidence and impact.
- Goal Runtime tracks whether the work is converging toward the target.

Goal Runtime does not execute patches directly. It produces planning and trace receipts.
