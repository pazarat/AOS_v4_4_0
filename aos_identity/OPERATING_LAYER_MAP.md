# Operating Layer Map

## Root operating container

- `operations_runtime/` — the parent runtime kernel and single operating spine. It calls all core and capability layers through registered APIs and produces Operational Insight.

## Identity

- `aos_identity/` — operating identity, constitution, and response laws.

## Core engines behind the Runtime Kernel

- `aos_core/intent/` — Intent Cognition.
- `aos_core/surface/` — surface awareness.
- `aos_core/entry/` — entry guards.
- `aos_core/solution_delivery/` — delivery from Operational Insight.

## Capabilities behind the Runtime Kernel

- `aos_capabilities/file_intelligence/` — artifact and file intelligence.
- `aos_capabilities/truth_runtime/` — truth grounding and sufficiency.
- `aos_capabilities/goal_runtime/` — goal ledger and experience memory when activated.
