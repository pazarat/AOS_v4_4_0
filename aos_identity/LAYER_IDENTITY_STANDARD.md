# Layer Identity Standard

Every mature AOS layer or capability should carry a short `LAYER_IDENTITY.md`.

The purpose is not documentation volume. The purpose is first-touch understanding: an external model must quickly learn what the layer is, what it is not, how it is used, what API it exposes, and how it relates to the operating spine.

## Required sections

1. Purpose
2. Not This
3. Public Interface
4. Internal Responsibilities
5. Boundaries
6. Relationship to AOS / Workshop / Active Project
7. Governance Rules
8. Failure Signals
9. When to Mature This Layer

## Rule

Layer identity explains operating philosophy. It must not hardcode project-specific payload names, temporary project paths, or sample project facts.
